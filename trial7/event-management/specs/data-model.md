# Data Model Specification

## Overview

The Event Management platform uses PostgreSQL 16 with Prisma ORM for data access.
The schema is multi-tenant with Row Level Security enabled on all tables.

See also: [api-endpoints.md](api-endpoints.md) for CRUD operations on these entities.
See also: [security.md](security.md) for RLS and data protection details.

## Prisma Service

- VERIFY: EM-DATA-001 — PrismaService extends PrismaClient with OnModuleInit and
  OnModuleDestroy lifecycle hooks for connection management

## Raw SQL Operations

- VERIFY: EM-DATA-002 — EventService.getEventStats uses $executeRaw with Prisma.sql
  template literal for safe parameterized queries (no $executeRawUnsafe)

## Row Level Security

- VERIFY: EM-DATA-003 — All tables have ENABLE ROW LEVEL SECURITY and FORCE ROW
  LEVEL SECURITY in migration SQL

## Entities

### User
- Fields: id (UUID), tenantId, email (unique), passwordHash, name, role, timestamps
- Relations: belongs to Tenant, has many Events, Tickets, AuditLogs
- Indexes: tenantId, email

### Tenant
- Fields: id (UUID), name, slug (unique), timestamps
- Relations: has many Users, Events, Venues, Categories, AuditLogs

### Event
- Fields: id, tenantId, title, description, startDate, endDate, status, price
  (Decimal 12,2), capacity, organizerId, venueId, categoryId, timestamps
- Relations: belongs to Tenant, User (organizer), Venue, Category; has many Tickets
- Indexes: tenantId, status, (tenantId, status), venueId, categoryId, organizerId
- Status enum: DRAFT, PUBLISHED, CANCELLED, COMPLETED

### Ticket
- Fields: id, tenantId, eventId, userId, price (Decimal 12,2), status, seatInfo, timestamps
- Relations: belongs to Event, User
- Indexes: tenantId, status, (tenantId, status), eventId, userId
- Status enum: RESERVED, CONFIRMED, CANCELLED, USED

### Venue
- Fields: id, tenantId, name, address, city, capacity, timestamps
- Relations: belongs to Tenant, has many Events
- Indexes: tenantId

### Category
- Fields: id, tenantId, name, description (optional), timestamps
- Relations: belongs to Tenant, has many Events
- Indexes: tenantId

### AuditLog
- Fields: id, tenantId, userId, action, entity, entityId, details (optional), createdAt
- Relations: belongs to Tenant, User
- Indexes: tenantId, userId, (tenantId, action)
- Action enum: CREATE, UPDATE, DELETE, LOGIN

## Conventions

- All monetary fields use Decimal @db.Decimal(12, 2) — never Float
- All models use @@map('snake_case_table_name')
- All enums use @@map('snake_case_enum_name') with @map on values
- UUID primary keys generated by @default(uuid())
- Timestamps: createdAt @default(now()), updatedAt @updatedAt
